# v0.6.0 — Catalog keys & computed recipe traits

Release date: 2026-07-12

## Summary

Phase 9 adds stable public keys for dishes and recipes, ingredient food groups, computed recipe traits derived from recipe ingredients, an ingredient taxonomy/resolver stack, and a taxonomy navigator UI. Weekly target matching and dish tags are unchanged — computed traits are additive metadata for display, filtering, and future scheduler migration.

## Added

### Catalog keys & computed traits

- **Dish public keys** — 32-character stable identifiers (`<slug>-<random>`), immutable on rename.
- **Recipe public keys** — up to 40-character identifiers (`<dish_public_key>-001`, sequence per dish; width grows for 4+ digit sequences).
- **`ingredients.food_group`** — controlled vocabulary with category→group mapping and optional override.
- **`recipes.computed_traits_json`** — family vector, food-group weights, vegan, carb-heavy, dominant carb/protein, etc.
- **API exposure** — `food_group`, `public_key`, `computed_traits_json` on catalog and meal-plan items.
- **Scheduler candidates** — `DishCandidate.computed_traits_json` (weekly targets still use tags).
- **Frontend** — ingredient food-group field; dish/recipe detail shows public key and trait summary.

### Taxonomy & ingredient resolver

- **Taxonomy YAML** — food groups, ingredient families, batch plan under `backend/mealroulette/data/taxonomy/`.
- **Ingredient seed** — single canonical file `backend/mealroulette/data/fixtures/mealroulette_ingredients_seed.yaml` (412 ingredients; validated against taxonomy on import). Removed duplicate `taxonomy/ingredients_seed.yaml` and embedded `ingredient_families` block from the fixtures file.
- **Browsing APIs** — food groups, families, ingredients by family, taxonomy overview.
- **Resolver** — exact/alias match, conservative fuzzy suggestions, classify-candidate with draft proposal.
- **Taxonomy navigator UI** — `/ingredients/taxonomy` (browse + resolver panel).

## Migration

- Alembic revision `022_computed_traits` — food groups, public keys, recipe sequences, computed traits backfill.
- `023_ingredient_taxonomy_metadata` — `storage_class`, `culinary_category`, `product_form`, `preservation` on ingredients.
- `024_ingredient_traits_storage` — `storage_after_opening`, `traits_json` on ingredients.
- `025_recipe_public_key_length` — widens `recipes.public_key` for long sequence suffixes.

## Unchanged

- Weekly target settings and `dish_matches_weekly_target` (tag-based).
- Shopping list aggregation.
- Telegram planning message format.
