## MealRoulette v0.2.0 — Manual Meal Planning

Second shippable release: plan lunch and dinner for the week, review what you actually ate, rate meals, and track lightweight leftovers — all from a mobile-friendly UI.

### What's included

**Weekly planning**
- Auto-scaffolded 7-day meal plan (lunch + dinner per day)
- Assign dishes and recipes manually; main recipe auto-selected when only one exists
- Lock/unlock slots to protect planned meals
- Week navigation with compact mobile layout

**Review flow**
- Separate **Plan** and **Review** tabs (review-first landing on mobile)
- Mark meals as **ate as planned**, **skipped** (optional reason/comment), or **ate leftovers**
- "Needs review" filter until each past/today slot is fully saved
- Star ratings per meal slot, linked to dish/recipe
- Undo status resets execution metadata and rating

**Lightweight leftovers**
- Reference a prior eaten meal as the leftover source (last 7 days, `eaten` status only)
- Or confirm **Unknown / same dish** without picking a source
- No portion inventory, fridge tracking, or leftover chains

**Backend**
- Planning API (`/api/planning/*`): current/week plan, assign, lock, status changes, ratings, reset
- Migrations **011–014**: meal plans, plan items, statuses, `meal_ratings`, `review_saved_at`
- Swagger OAuth2 token endpoint for `/docs` Authorize
- Integration and unit tests for planning rules and API

**Mobile polish**
- Bottom tab bar with safe-area padding
- Larger touch targets on meal actions and star ratings
- Keyboard-accessible star rating control

### Database migrations

Applies Alembic revisions **011–014** on top of v0.1. Run via Docker:

```bash
docker compose up -d --build
```

### Not in this release

Shopping lists, Telegram reminders, automatic scheduler/reroll, leftover inventory, LLM-assisted entry, and cooking mode — planned for v0.3+.

### Upgrade from v0.1.0

```bash
git checkout v0.2.0
docker compose up -d --build
```

Existing dish library data is preserved; new tables are created by migrations. Bootstrap admin only if this is a fresh database:

```bash
docker exec -it mealroulette-api python -m mealroulette.commands.bootstrap_admin \
  --username admin --email admin@example.com
```

See [README.md](https://github.com/didac-crst/mealroulette/blob/v0.2.0/README.md) for environment setup.
