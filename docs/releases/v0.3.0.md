## MealRoulette v0.3.0 — Shopping Lists & Ingredient Catalog

Third shippable release: generate a shopping list from your meal plan, aggregate quantities intelligently across units, and manage ingredient conversions from an admin dashboard.

### What's included

**Shopping lists**
- Preview shopping list for the next N days from planned meals (lunch + dinner)
- Optional **save** list to track checked-off items
- Group items by grocery category
- Exclude **pantry items** (per-ingredient flag)
- Show **planned meals** in the window before the list
- Per-item **source breakdown** — which meal contributed each quantity
- Approximate totals (`~860 g`) when mass/count merge via approved conversions, with `includes:` component lines

**Unit aggregation**
- Strategy-aware merging in `services/quantities` (`prefer_mass`, `prefer_count`, `allow_approximate_conversion`, etc.)
- Same-dimension merge through base units (g + kg)
- Cross-dimension merge only with **approved** ingredient-specific conversions
- Incompatible lines stay separate

**Ingredient catalog**
- YAML seed: `mealroulette_ingredients_seed.yaml` (~100+ household ingredients, aliases, units)
- `import_ingredient_seed` CLI — idempotent import with optional bootstrap approval
- Per-ingredient **unit behavior**: family, preferred shopping unit, aggregation unit/strategy
- Ingredient unit conversions with `confidence`, `source`, `approved` metadata
- Conversions CRUD API with unique `(ingredient, from_unit, to_unit)` constraint

**Admin UI**
- `/ingredients` — browse catalog
- `/ingredients/:id/edit` — aliases, conversions, unit behavior, pantry flag
- Approve/reject conversion suggestions for shopping aggregation

**Dev tooling**
- Recommended bootstrap: `import_ingredient_seed` → `import_sample_dishes`
- Legacy `reference/ingredient_conversions.yaml` deprecated (conversions live in ingredient seed)

**Backend**
- Shopping API: preview, create, fetch, check-off items
- Migrations **015–018**: shopping lists, per-meal contributions, ingredient unit behavior, conversion uniqueness
- 72 automated tests

**Frontend**
- `/shopping` page (nav tab **List**)
- Ingredient list and edit pages for admins

### Database migrations

Applies Alembic revisions **015–018** on top of v0.2. Run via Docker:

```bash
docker compose up -d --build
```

### Bootstrap catalog data

After migrations and admin bootstrap:

```bash
docker exec -it mealroulette-api python -m mealroulette.commands.import_ingredient_seed
docker exec -it mealroulette-api python -m mealroulette.commands.import_sample_dishes
```

### Not in this release

Telegram reminders, automatic scheduler/reroll, leftover inventory, LLM-assisted entry, cooking mode, JSON backup/restore, and multilingual content (design in [LOCALIZATION.md](../LOCALIZATION.md), implementation Phase 11).

### Upgrade from v0.2.0

```bash
git checkout v0.3.0
docker compose up -d --build
docker exec -it mealroulette-api python -m mealroulette.commands.import_ingredient_seed
docker exec -it mealroulette-api python -m mealroulette.commands.import_sample_dishes
```

Existing dishes and meal plans are preserved; new tables and columns are created by migrations.

See [README.md](https://github.com/didac-crst/mealroulette/blob/v0.3.0/README.md) for environment setup.
