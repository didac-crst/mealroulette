# v0.8.0 ‚Äî Taxonomy hardening and backup

Release date: 2026-07-13

## Summary

Phase 11 hardens ingredient taxonomy before freezing the backup contract. It adds first-class taxonomy tables, moves weekly target matching toward computed recipe traits, introduces meal composition metadata, and ships admin backup/export/import with scheduled backups and restore documentation.

## Added

### Taxonomy hardening

- First-class `food_groups` and `ingredient_families` database tables.
- `ingredients.family_id` foreign key with seed and legacy-family backfill.
- Canonical ingredient enforcement for taxonomy family IDs.
- Taxonomy seed helper for keeping YAML taxonomy and database tables aligned.

### Scheduler target matching

- Weekly targets use main-recipe computed traits first for derivable targets.
- Legacy protein/carb tags remain a documented temporary fallback when computed traits are missing.
- Non-derivable styles such as soup remain curated style classifications.

### Meal composition

- Dish-level `meal_composition` metadata.
- `simple_dish_part` for simple dishes.
- Course derivation aligned with meal composition.
- Updated dish edit UI and classification summaries.

### Backup, export, and import

- Admin-only full JSON export and import.
- Backup settings and backup run tracking.
- Manual and scheduled backup execution from the worker.
- Retention cleanup for backup files under `/backups`.
- Optional `pg_dump` backup with timeout and failure reporting.
- Admin backup settings UI.
- Restore runbook in [RESTORE.md](../RESTORE.md).

## Migrations

- `027_dish_meal_composition` ‚Äî dish meal-composition metadata.
- `028_taxonomy_tables` ‚Äî food groups, ingredient families, and `ingredients.family_id`.
- `029_backup_tables` ‚Äî backup settings and backup runs.
- `030_taxonomy_family_backfill` ‚Äî backfill ingredient family IDs from seed YAML and legacy aliases.

```bash
docker compose exec api alembic upgrade head
```

Restart **api** and **worker** after upgrading.
