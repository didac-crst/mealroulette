## MealRoulette v0.5.0 — Explainable Scheduler

Fifth shippable release: automatic weekly meal roulette with family-vector similarity, human-readable selection reasons, plan UI controls, and optional scheduled generation with Telegram “New roulette”.

### What's included

**Scheduler engine**
- Family-vector similarity built on the fly from main-recipe ingredients (L1-normalized family proportions, cosine distance)
- Soft scoring: seasonality, ratings, weekly tag targets with tolerance, temporal neighbour penalties (eaten + locked/planned + in-run picks)
- 50-attempt week generator; hard excludes inactive dishes and same-dish window violations
- Full rules: [docs/SCHEDULER.md](../SCHEDULER.md)

**Roulette actions (Plan UI + API)**
- **Generate week** — fill all unlocked, auto-eligible slots; preserve locked and manual picks
- **Reroll** — replace one today/future slot
- **Undo** — restore last generate or reroll (one level)
- **Swap** — exchange dish/recipe between two planned slots (no scoring)
- **Plan for…** — assign a dish from the gallery to a calendar lunch/dinner slot
- Selection reasons and variety summary on generate/reroll details

**Scheduled roulette**
- Worker minute poll (alongside daily Telegram reminder)
- Configurable weekday, time, timezone, target week offset
- Optional **“New roulette”** Telegram broadcast after successful generate
- Admin **Run now** (`POST /api/scheduler/run-roulette`)

**Admin API & UI**
- `GET/PUT /api/scheduler/settings` — schedule and notify flags
- `POST /api/scheduler/run-roulette` — manual scheduled run
- **Settings hub** (`/settings`, mobile **More** tab): weekly targets, auto roulette, Telegram, ingredients
- `/settings/targets` — edit weekly fish/meat/pasta/rice/vegetarian targets
- `/settings/scheduler` — auto roulette schedule; household time shown in settings
- Mobile nav: Plan · Review · List · Dishes · **More** (admin); desktop **Settings** link

**Review hardening (PR #8)**
- Weekly target scoring counts already-assigned dishes from the full catalog
- Scheduled job runs at or after `run_time` on the configured weekday (not only the exact minute)
- Symmetric same-dish window; reroll never picks the same dish; locked meals cannot be swapped
- Plan row lock (`SELECT FOR UPDATE`) on generate/reroll; undo cleared on manual assign/swap/lock
- Household calendar “today” uses scheduler timezone (not server UTC)

**Planning rules**
- `planning_rules` table with active weekly targets and tolerances (seeded defaults)
- Admin API from slice 1 (`GET/PUT /api/planning-rules`)

**Backend**
- Migrations **020** (`planning_rules`, `scheduler_settings`) and **021** (`meal_plans.last_roulette_undo_json`)
- `services/scheduler/` — vectors, scoring, generator, variety assessment
- 153 automated backend tests (including Phase 8 acceptance API tests)

### Configuration

No new required environment variables. Scheduler uses existing Telegram settings when `notify_telegram` is enabled.

As admin, open **Settings** (mobile: **More**) → **Weekly targets** or **Auto roulette** to configure targets and the Friday job.

**Local dev:** leave `VITE_API_URL` empty; the UI proxies `/api` to the backend (works on `localhost` and LAN IP without hardcoding).

### Database migrations

Applies Alembic revisions **020** and **021** on top of v0.4. Run via Docker:

```bash
docker compose up -d --build
```

Ensure dish fixtures and planning rules are present (default seed on first migrate).

### Not in this release

Composable meals (complete + two halves per slot), LLM-assisted entry, leftover inventory, cooking mode, JSON backup/restore, bot pairing codes, multilingual content.

### Upgrade from v0.4.0

```bash
git checkout v0.5.0
docker compose up -d --build
```

Open **Plan**, use **Generate week** on a future week, or enable scheduled roulette in **Scheduler** settings.

See [README.md](https://github.com/didac-crst/mealroulette/blob/v0.5.0/README.md) for full setup.
