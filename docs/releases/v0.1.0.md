## MealRoulette v0.1.0 — Foundation

First shippable release: a self-hosted stack for managing your household dish library.

### What's included

**Platform**
- FastAPI backend + PostgreSQL + React frontend, orchestrated with Docker Compose
- Health checks, CI (backend + frontend tests), pre-commit hooks

**Authentication**
- JWT login with access/refresh tokens
- Admin and regular user roles
- Protected API routes and frontend session restore

**Catalog API**
- Dishes, recipes, ingredients, units, and tags
- Recipe steps and recipe ingredients with quantity/unit validation
- Ingredient resolve/confirm flow for fuzzy name matching
- YAML-backed reference data (units, tags) with idempotent seeding

**Dish library UI**
- Login and authenticated shell
- Dish card grid (optional image URL or course emoji)
- Dish create/edit with classification tags, seasonality, and planning flags
- Recipe create/edit with steps, ingredients, and main-recipe selection
- Dish detail with recipe variants; times and difficulty derived from the main recipe

**Data model highlights**
- Dish course: starter, main, dessert
- Tags: protein, carb, style, temperature
- Seasonality: all-year or seasonal with preferred months
- Main recipe drives displayed prep/cook time and difficulty

### Database migrations

Applies Alembic revisions **005–010** on top of earlier schema (004 is a no-op placeholder for existing histories). Run via Docker:

```bash
docker compose up -d --build
```

### Not in this release

Meal planning, shopping lists, Telegram reminders, automatic scheduler, LLM-assisted entry, dish search/filters, and dietary inference from ingredients — planned for v0.2+.

### Upgrade path

Fresh clone:

```bash
git checkout v0.1.0
docker compose up --build
```

See [README.md](https://github.com/didac-crst/mealroulette/blob/v0.1.0/README.md) for environment setup and default credentials.
