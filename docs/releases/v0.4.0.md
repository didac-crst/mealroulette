## MealRoulette v0.4.0 ‚Äî Telegram Reminders

Fourth shippable release: configure a Telegram bot for daily shopping reminders, on-demand planning/shopping messages, and tap-to-open full recipes from the meal plan.

### What's included

**Telegram bot (household)**
- `/subscribe` / `/unsubscribe` ‚Äî join or leave the reminder list (no fixed chat ID in settings)
- `/planning [days]` ‚Äî HTML meal plan (default 3 days) with prep/cook times
- `/reminder [days]` ‚Äî planning block + per-ingredient shopping breakdown
- `/shopping [days]` ‚Äî category-grouped shopping totals only
- `/help` ‚Äî command summary
- **Recipe links** in planning/reminder ‚Äî tap a dish to receive ingredients + steps as HTML (`/start recipe_<id>`)

**Scheduled reminders**
- APScheduler worker polls settings every minute and sends at `daily_reminder_time` (timezone-aware)
- Same HTML message as **`/reminder`** (planning + ingredient breakdown, pantry included)
- Window length = `shopping_window_days` setting (default 3)
- `last_sent_at` / `last_error` on settings row

**Admin API & UI**
- `GET/PUT /api/telegram/settings` ‚Äî reminder schedule (no secrets in response)
- `GET /api/telegram/subscribers` ‚Äî who is subscribed
- `POST /api/telegram/test` ‚Äî connectivity test
- `POST /api/telegram/send-daily-reminder` ‚Äî manual run (same path as worker)
- `POST /api/shopping-lists/{id}/send-telegram` ‚Äî send a saved list
- `/settings/telegram` ‚Äî admin settings page (save, test, send now, subscriber list)

**Ingredient catalog polish**
- `ingredient_categories.yaml` reference list + `GET /api/ingredient-categories`
- Category dropdown in ingredient editor
- Seed import backfill for uncategorized dish-imported ingredients (aliases)

**Branding**
- Optimized logo assets (`frontend/public/`) ‚Äî favicon, header, Telegram icon

**Backend**
- Migration **019**: `telegram_settings`, `telegram_subscribers`
- Bot token via `TELEGRAM_BOT_TOKEN` in `.env` (API + worker); optional `TELEGRAM_BOT_USERNAME` for recipe deep links
- 99 automated backend tests

### Configuration

Add to `.env`:

```bash
TELEGRAM_BOT_TOKEN=your-token-from-botfather
TELEGRAM_BOT_USERNAME=your_bot_username   # optional; enables dish links in planning
```

Restart api + worker, then in Telegram send `/subscribe` to your bot. As admin, open **Telegram** in the app header to enable reminders and send a test.

### Database migrations

Applies Alembic revision **019** on top of v0.3. Run via Docker:

```bash
docker compose up -d --build
```

Re-import ingredient seed if categories were missing:

```bash
docker exec -it mealroulette-api python -m mealroulette.commands.import_ingredient_seed
```

### Not in this release

Automatic scheduler/reroll, leftover inventory, LLM-assisted entry, cooking mode, JSON backup/restore, bot pairing codes, multilingual content.

### Upgrade from v0.3.0

```bash
git checkout v0.4.0
docker compose up -d --build
```

Add `TELEGRAM_BOT_TOKEN` to `.env`, subscribe in Telegram, enable reminders in admin settings.

See [README.md](https://github.com/didac-crst/mealroulette/blob/v0.4.0/README.md) for full setup.
