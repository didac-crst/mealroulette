# v0.10.0 - Composable meals, simple dishes, and faster roulette

Release date: 2026-07-14

## Summary

This release changes the planner from "one slot equals one dish" to a real meal-slot model. Lunch and dinner slots can now contain multiple dish lines: roulette may choose either one complete main dish or a simple pair made from a centerpiece plus a side, while household members can manually add extras such as sides, desserts, or additional dishes.

It also imports a large simple-dish catalog, hardens fixture imports so bad catalog data fails early, and makes week generation fast enough for interactive use with a larger centerpiece/side catalog.

## Added

### Composable meal slots

- New `meal_plan_item_dishes` table for dish lines inside a meal slot.
- Meal slots now act as containers with zero or more dish lines.
- Roulette-generated lines and manually added lines are distinguished by source.
- A slot can be marked `Do not plan`, preventing roulette from filling known-away meals.
- Existing single-dish slots are migrated into one dish line for compatibility.
- Meal titles and status formatting now handle multi-dish slots.

### Simple dishes and fixture data

- Added a large `simple_dishes.yaml` fixture with centerpiece and side dishes.
- Expanded canonical ingredient seed data so the fixture can import cleanly.
- Importers now reject duplicate canonical ingredients, alias clashes, and recipe lines that do not resolve to canonical ingredients or approved aliases.
- Added helper scripts for repairing simple-dish fixture instructions and sorting ingredient seed data.

### Roulette pairing and performance

- Week generation can choose either:
  - one `main_dish`; or
  - one `centerpiece` plus one `sidedish`.
- Candidate scoring is precomputed once per slot, then reused for pair scoring.
- Centerpiece/side exploration uses a bounded but diverse shortlist instead of scoring the full cartesian product.
- Large simple-dish catalogs automatically reduce generation attempts to keep planning interactive.
- Added `backend/scripts/benchmark_generate_week.py` for manual performance checks.

### Meal composition analytics

- Meal-level composition is derived from all dish lines in the slot.
- Planning UI can show composition charts for combined meals.
- Scheduler, shopping, and planning serialization now understand multi-line slots.

### Parallel backend testing

- Added `pytest-xdist` support.
- Added per-worker test database setup with `backend/scripts/setup_test_databases.py`.
- Added `make test-backend-parallel`.
- Split CI into backend unit, backend integration, frontend unit, frontend build, and taxonomy validation jobs.

## Changed

- Planning APIs now expose line-level create, update, delete, and assignment behavior.
- Shopping-list generation aggregates ingredients from every dish line in a meal slot.
- Undo snapshots preserve composable meal lines.
- Scheduler neighbor/history logic treats paired simple dishes as one meal slot where appropriate.
- Frontend week generation timeout is 60 seconds as a safety margin, though normal generation is expected to complete much faster.
- `Do not plan` respects locked-slot rules.
- Duplicate line positions are rejected with a conflict instead of surfacing as an unhandled database error.
- Updating a meal line to a different dish resolves the new dish's default recipe unless a recipe is explicitly provided.
- Deleting a meal line reindexes remaining lines safely.

## Data and migration notes

- New migration: `031_composable_meal_lines`.
- Fresh database setup and upgraded databases both create line rows for existing planned dishes.
- Legacy `meal_plan_items.dish_id` and `meal_plan_items.recipe_id` remain for compatibility during the transition, but composable behavior is represented by child dish lines.
- Fixture imports are stricter than before; catalogs with non-canonical ingredients or alias collisions must be fixed before import.

- Frontend production build passed.
- Taxonomy validation passed with 434 accepted ingredients and 0 blockers.

## Upgrade notes

- Run Alembic migrations before starting the app after pulling this release.
- If loading fixtures from scratch, import canonical ingredients before recipe/simple-dish fixtures.
- Existing meal plans keep their visible behavior, but planned dishes are now represented as line records under each slot.
- Large simple-dish catalogs should no longer make Generate Week time out, but benchmark locally if the catalog grows substantially.
